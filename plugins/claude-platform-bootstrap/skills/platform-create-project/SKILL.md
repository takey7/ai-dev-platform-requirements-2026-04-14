# Platform Create Project

## Workflow

1. Prefer `platform create-project "<Project Name>"`.
2. Require that the user has already configured GitHub login, Claude login, Codex login, and `ATLASSIAN_API_TOKEN`.
3. Treat `create-project` as the greenfield convenience workflow; do not replace `bootstrap` as the canonical repo-level primitive.
4. Let the CLI generate repo name, Jira key, and Confluence space by default unless the user explicitly overrides them.

## Defaults

- Keep Jira on the Kanban template.
- Keep GitHub repos private.
- Keep launch mode at `tmux`.
- Preserve the repo-local Atlassian scope guardrails generated by the baseline.
- Later Jira issue creation should stay explicit-only and inside the repo's generated Jira project key.
